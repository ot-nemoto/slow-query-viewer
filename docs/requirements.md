# 要件定義

## 機能要件

### ファイル管理

- 単一または複数のスロークエリログファイルを同時アップロードできる
- ドラッグ＆ドロップによるファイルアップロードに対応する
- アップロード済みファイルの一覧（ファイル名・サイズ・エントリ数）を表示する
- 個別ファイルの削除と全ファイル削除ができる
- 不正なファイルはエラー通知で処理し、正常なファイルの解析は継続する

### ログ解析

- MySQL スロークエリログ形式を解析する（Time, User@Host, Query_time, Lock_time, Rows_sent, Rows_examined, use 文, クエリ本体）
- AWS RDS 形式のログバリアントにも対応する
- 不正なエントリはスキップし、有効なエントリの解析を続行する

### クエリ正規化・グルーピング

- 数値リテラルを `?`、文字列リテラルを `'?'` に置換してクエリを正規化する
- 正規化されたクエリ単位でグルーピングする
- パラメータ値別の詳細分析（実行回数、平均/最大/最小/合計実行時間）を提供する

### 統計サマリー

- 総クエリ数、総実行時間、平均実行時間、最大実行時間、最小実行時間を表示する
- 解析期間（開始日時〜終了日時、期間日数）を表示する
- 最も遅いクエリの詳細（実行時間、タイムスタンプ、ユーザー、検査行数、SQL）を表示する

### 結果テーブル

- 正規化クエリ、実行回数、総実行時間、平均実行時間、最大実行時間、最小実行時間を列として表示する
- 各数値列でソート（昇順/降順）できる
- 上位 20 件を表示する
- 各クエリの「分析」ボタンからパラメータ分析モーダルを開ける

### 時系列チャート

- クエリ実行時間の時系列変化を折れ線グラフで表示する
- ファイルごとに異なる色で描画し、ファイル単位で表示/非表示を切り替えられる
- ツールチップでファイル名、実行時間、ユーザー、検査行数を確認できる

### 通知

- 処理結果をトースト通知（成功/警告/エラー）で表示する
- 5 秒後に自動消去される

## 非機能要件

- **デプロイ**: GitHub Pages への静的デプロイ（サーバー不要）
- **処理方式**: すべての解析はクライアントサイド（ブラウザ内）で完結する
- **レスポンシブ**: モバイル・デスクトップ対応
